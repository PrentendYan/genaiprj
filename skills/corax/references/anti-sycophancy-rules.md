# CORAX Anti-Sycophancy Rules

CORAX uses same-family Codex producer and reviewer roles, so anti-sycophancy controls are mandatory.

## Core Controls

1. Blind brief: the reviewer sees stripped facts, not producer conclusions.
2. Independent context: reviewer runs in an ephemeral read-only workspace.
3. Forced skepticism: reviewer must provide counterarguments and alternatives.
4. Suspicious perfection: all-pass results are treated as a risk signal.
5. Claude Sentinel: a heterogeneous model checks for Codex-on-Codex groupthink.

## Reviewer Requirements

- Start from the assumption that a subtle bug may exist.
- Cite evidence from the blind brief.
- Distinguish confirmed bugs from missing evidence.
- Do not reward persuasive writing without proof.
- Flag ambiguous taxonomy boundaries.

## Quant Failure Modes

- Lookahead or target leakage.
- Full-sample normalization.
- Random splits on time series.
- Missing transaction costs.
- Unsupported performance claims.
- Cherry-picked OOS windows.
- Weak or absent baselines.
