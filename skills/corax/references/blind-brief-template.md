# CORAX Blind Brief Template

The blind brief is the only content the Codex Reviewer should see.

## Keep

- Task and phase objective.
- Data sources, dates, assets, and filters.
- Code snippets, changed files, and commands.
- Metrics, tables, logs, and test results.
- Automated audit output.
- Known constraints and open questions.

## Strip

- Producer recommendations.
- Producer confidence statements.
- Claims that tell the reviewer the answer.
- Subjective wording.
- Marketing language.
- Any statement that cannot be traced to evidence.

## Template

```markdown
# Blind Brief

## Phase

## Task

## Evidence

## Code / Artifacts

## Metrics and Tests

## Automated Audit Results

## Constraints

## Open Questions
```

## Self-Check

Before review, scan for language that implies conclusion, superiority, success, recommendation, or certainty. Remove it unless it is quoted evidence being audited.
