# Blind Brief Template

Use this template to strip producer conclusions before adversarial review.

## Include

- Task statement.
- Data sources, dates, assets, and filters.
- Code snippets or changed files.
- Metrics, tables, logs, and test results.
- Stated assumptions and constraints.
- Known failures or warnings.

## Exclude

- Producer verdicts such as "this works" or "this is best".
- Recommendations and subjective confidence language.
- Marketing language.
- Claims without direct evidence.
- Any text that tells the reviewer what conclusion to reach.

## Output

```markdown
# Blind Brief

## Task

## Evidence

## Code / Artifacts

## Metrics

## Constraints

## Open Questions
```
