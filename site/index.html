The Question

AI tools can write a backtest in minutes, and that speed is exactly what makes research mistakes easier to miss. A shift in the wrong direction, a normalization step before the split, a random shuffle of a time series, a Sharpe ratio computed gross of costs: none of these errors announces itself, and each produces a backtest that looks profitable.

This project asks whether CORAX, a Codex reviewer with a blind brief and a configurable second agent, can catch those mistakes while avoiding producer-framing bias.

The Benchmark

Forty-five labeled audit cases, each a small piece of real research material, drawn from BTC data, multi-stock equity data, and real tutorial notebooks. Five failure modes are covered, alongside eleven clean controls written as deliberate near-misses, so an over-eager reviewer is exposed as precisely as a careless one.

Three Findings

01

A live reviewer followed the data flow

The key case declares a transaction-cost variable, then never subtracts it from returns. CORAX live review flags the missing cost application instead of being fooled by the word cost.

02

The blind brief changed behavior

When the producer claim was visible, the reviewer added an extra unsupported-claim false positive. After stripping the claim, the reviewer produced the cleaner issue set.

03

The second agent is explicit

The repo now supports a same-family Codex meta-reviewer and a later Claude Sentinel path. Both produce artifacts and gate decisions.

Where AI Stands

Where it helps

Semantic review can catch code-path bugs that keyword scanners miss, and blind briefs reduce producer-framing effects.

Where it does not

A one-case smoke run is evidence of feasibility, not a final statistical claim. The selected-case ablation still needs a full run.

Where a human is needed

Finance taxonomy still matters. Ambiguous shifts, cost timing, and label boundaries need human adjudication.
