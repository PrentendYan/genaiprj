Can an AI agent audit quant research?

A one-line error can turn a weak strategy into a persuasive chart. We built CORAX, an adversarial AI audit workflow, and ablated it component by component to find out what actually makes an automated reviewer reliable.

The headline result
0.60 ───▶ 0.86

Stripping the producer's self-promotional framing before review raised audit F1 by 26 points. Adding a second AI agent on top did not raise it further; instead it changed how cautiously the system gates borderline work. Framing control mattered more than a second opinion.

1 The problem

The failure modes an automated auditor has to catch.

Quant research is fragile. The same handful of mistakes show up again and again in backtests and in AI-generated research code, and each one can make a strategy look better than it is:

  • lookaheadbuilding features from future returns or a negative shift
  • normalization_leakagenormalizing across the full sample before the train/test split
  • temporal_splitapplying a random shuffle split to time-series data
  • missing_costsreporting returns and Sharpe before deducting trading costs
  • unsupported_claimconcluding outperformance with no baseline or robustness check

CORAX treats an AI reviewer as a research auditor: it reads a submitted artifact, emits structured findings, and those findings are scored against human labels.

2 How CORAX works

Separate the producing, the reviewing, and the reviewing-of-the-review.

The core idea is to structurally separate production, review, and meta-review, so the reviewer is not swayed by the author's language and no single agent decides an audit alone.

The blind brief

Before the reviewer sees anything, an independent module rewrites the submission to describe only the code facts: every "remarkable Sharpe" or "outperforms any baseline" is removed, so the reviewer judges the code, not the sales pitch.

The second agent

An optional meta-reviewer inspects the reviewer's output for groupthink, missed concerns, and over-confidence. It comes either from the same model family (Codex) or a different one (Claude Sentinel), the latter giving a genuinely independent perspective.

3 The benchmark

Nine cases, every issue type, plus traps.

The ablation runs on nine human-labeled cases drawn from real fixtures: a BTC sample, a QuoteMedia stock sample, and two tutorial notebooks. They cover every issue type and deliberately include clean near-miss controls (code that looks suspicious but is correct) and one boundary case whose label is genuinely ambiguous, to test whether the reviewer is fooled by surface patterns. Submitted code and answer labels are stored separately, so the reviewer never sees the key.

4·5 The ablation & results

Toggle one component at a time; read each component's contribution off the gap.

CORAX has two switchable components: the blind brief and the second agent. Four configurations form a cumulative ladder, where every step changes exactly one thing, so each performance gap is that component's net contribution. All four use the same reviewer model, cases, labels, and scoring code. A deliberately weak, cheap model is used throughout: it keeps a multi-agent experiment affordable to re-run, and it turns the study into an architectural stress test.

Finding 1

The F1 gain comes almost entirely from the blind brief

From single_llm to blind_only, F1 jumps 0.60 → 0.86, and the only thing that changed is the blind brief. Recall never moves: the brief makes the reviewer neither find nor miss any real bug. The entire gain is precision: false positives collapse from 7 to 1. All six eliminated false positives were unsupported_claim errors triggered by the producer's bragging. The brief works not by making the reviewer smarter, but by removing the input that misled it.

Finding 2

The second agent doesn't change F1, it changes the gate

Adding a second agent leaves precision, recall, and F1 untouched at 0.86. What it changes is where each case is routed. Without a second agent, the system only ever says PASS or FAIL. The cross-family Claude Sentinel, free of Codex's blind spots, escalates far more borderline cases to human review instead of letting a weak reviewer decide alone.

7 What AI can do / what humans must do

▷ What AI can do

Act as a high-throughput first-pass auditor: read code, writeups, and notebook fragments, and flag suspicious patterns. It goes beyond keyword matching: in the cost case it followed the data flow to catch a semantic implementation error, narrowing where human reviewers need to look.

◆ What humans must do

Decide whether flagged code is actually wrong in a financial setting; define which mistakes count as failures versus defensible conventions; build and validate the benchmark and its near-miss controls; and interpret findings, where a model flag is evidence for review, never proof that a strategy is valid or invalid.
